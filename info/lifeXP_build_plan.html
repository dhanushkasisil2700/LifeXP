A prompt-by-prompt construction guide for Claude Code. Each sprint is a self-contained unit: one focused prompt, a clear goal, normal cases, edge cases, and a pass/fail checkpoint before moving on.

How to use this plan with Claude Code

  1. One sprint at a time. Copy the exact prompt text into Claude Code. Do not combine sprints — each builds on the last.
  2. Run all test cases before proceeding. Every sprint has Normal, Edge, and Boundary tests. All must pass before you move to the next sprint.
  3. Read the Checkpoint. The checkpoint is your go/no-go gate. If anything fails, ask Claude Code to fix it within the same sprint — do not advance.
  4. Keep a tests/ folder. Ask Claude Code to write pytest tests for each sprint. They accumulate. Run the full suite at every Phase boundary.
  5. At phase boundaries, run the full suite. Before starting Phase 2, all Phase 1 tests must be green. Same for every subsequent phase.
